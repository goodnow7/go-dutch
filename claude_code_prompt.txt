## 회비를 회원별로 정산하는 프로그램을 짜줘

- 회원 명단을 입력받아. 모임명, 시작 날짜(YYYYMMDD), 종료 날짜(YYYYMMDD), 회원 이름(n명)들을 입력받으면 돼
- 입력받고 확인 버튼을 누르면 사용 내역을 입력하는 화면이 나옴
- 날짜, 건별 항목명, 건별 금액, 결제회원, 적용회원 선택하면 적용회원당 1/n 금액 분배된 결과. 건별 메모 입력란도 만들어. 
- 결제회원: 건별 항목을 지불한 회원. 
- 적용회원: 결재된 항목을 비용을 적용받는 회원. 일부 회원은 결제 항목에 해당 안될 수 있으므로 구분해야 함. 기본값은 처음 입력받은 회원 이름 모두 체크됨. 결제회원이 적용회원이 될 수도 있고 아닐 수도 있음. 
- 분배금액: 결제건의 지불금액을 적용회원별로 나눈 금액
- 맨 아래에는 회원별 적용금액의 합계 계산, 건별 금액의 총액과 적용회원별 총 합계 금액이 맞는지 보여주기.

- 등록된 회원 삭제 기능 적용.  단, 삭제할 경우는 이전 모임 자료에는 그대로 남아 있어야 해. 이를 위해 회원관리 메뉴를 첫 화면에 배치하는 것도 좋아. 

- 모임별 요약 페이지 구성. 모임명, 시작날짜, 종료날짜, 총금액 리스트 목록
- 모임 만들기 할 때 등록된 회원을 선택할 수도 있고, 추가할 수도 있게 모임 등록 화면을 수정해.  다른 모임을 나중에 만들 때에도 저장된 회원들을 불러올 수 있어야 해.

## UI
- 회원 목록은 이름을 입력하고 엔터를 누르면 다음 회원 입력 받는 대기로 전환시켜
- 회원 목록은 이름을 입력하고 '회원추가' 버튼을 누르면 다음 회원 입력칸에 커서가 입력 대기 상태가 되도록 해.
- 날짜 입력은 달력 아이콘을 추가하고, 아이콘은 흰색으로 잘 보이게 해.
- 사용내역 입력시 날짜는 기본적으로 사용 내역 화면을 오픈하는 날짜로 설정해.  
- 이용내역 전체 나열 순서는 가장 최근 내역이 맨 위로 배치되게 세팅해.
- 별도 창의 경우 닫기 x 버튼은 항상 창의 타이틀 우측 끝에 위치해. 특히 모바일에서도 우측 끝을 유지하도록 해.
